- Symptom: a route started throwing `TypeError: Cannot read properties of undefined (reading 'upsert'|'findUnique'|...)` on a Prisma model delegate such as `prisma.userSyncState`, even though `prisma/schema.prisma` and the generated client on disk both already included that model.
- Root cause: the dev server kept reusing a cached global Prisma client instance that was created before the new model existed. Hot reload picked up the new route code, but `lib/prisma.ts` still returned the old singleton, so newly added delegates were missing at runtime.
- Fix: version the cached Prisma singleton with a signature derived from the generated client's model list (and Prisma client version), and recreate the singleton when that signature changes or expected delegates are missing.
- Guardrail: if a Prisma delegate is `undefined`, treat it as a stale client/runtime mismatch before assuming the database table is missing. Missing tables usually surface as Prisma query errors, not JavaScript property-access errors.
