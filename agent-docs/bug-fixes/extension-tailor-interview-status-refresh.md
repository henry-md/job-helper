- Symptom: when Tailor Resume was started from the Chrome extension shortcut, Step 1 could reach `needs_input` in the backend, but the side panel only showed `Resume questions are waiting in the side panel.` and never rendered the interview card or textarea.
- Root cause: the extension reloaded personal info only when `lastTailoringRun.capturedAt` changed. Background updates keep the same `capturedAt` while a single run transitions from `running` to `needs_input`, so the side panel never re-fetched the pending interview payload after the question was created.
- Fix: make the side panel refresh personal info when the persisted last-run status or tailored-resume id changes, not just when a brand-new run timestamp appears.
- Guardrail: extension UI that derives live state from `chrome.storage.local` should treat status transitions on the same run as meaningful refresh triggers, especially when the backend promotes a run into an interactive interview state.
