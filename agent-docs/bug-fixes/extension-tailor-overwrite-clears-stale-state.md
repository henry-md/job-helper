- Symptom: after choosing `Overwrite` on an existing Tailor Resume run in the extension, the side panel could still show the old interview thread and previous saved-run details while the new run was already starting.
- Root cause: the extension changed `captureState` to `running`, but kept the old `tailorInterview`, `lastTailoringRun`, and `personalInfo.activeTailoring` hydrated until the overwrite request finished, so stale UI leaked through during the transition.
- Fix: when overwrite starts, immediately clear the local interview thread and finish prompt state, replace the last-run card with a fresh running record for the new attempt, dismiss the existing-tailoring prompt, and ignore stale `personalInfo` active-tailoring hydration until the overwrite resolves.
- Guardrail: when the user explicitly replaces one in-flight extension workflow with another, clear extension-local state optimistically at the handoff instead of waiting for the backend round-trip to reconcile old and new runs.
