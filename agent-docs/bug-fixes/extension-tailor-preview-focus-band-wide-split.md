# Tailor Resume preview magnifier width and zoom stability

- Symptom: in the extension wide split view, the PDF spotlight could stay visually trapped near the preview pane instead of spanning the full side panel. In the web review modal, the same spotlight could become too wide and cover too much of the dashboard. During hover/scroll, tiny vertical movement could also make the spotlight zoom jump in and out.
- Root cause: spotlight width was derived from PDF-local geometry instead of the owning review surface. A later full-width version still computed hover zoom from the nearest text line, so moving from a long bullet line to a shorter line or heading changed the scale abruptly.
- Fix: make `TailoredResumeInteractivePreview` own the magnifier geometry with an explicit width policy. The web app uses the default `magnifierMaxWidthRatio={0.6}` cap; the extension passes `magnifierMaxWidthRatio={null}` so the spotlight can use the full side-panel width. Magnifier state carries explicit `width`, `height`, and `zoom`; hover zoom is based on the stable page text-column width while vertical crop still follows the nearest line.
- Guardrail: do not derive hover zoom from the nearest line width. Keep hover zoom stable for the page/column so scrolling changes only the vertical crop, not the scale. Verify both focus-click and pointer-hover states in a side-panel-width surface, including hover directly over clickable segment targets, and assert that `backgroundSize` stays stable while nearby hover positions or small scrolls change `backgroundPosition`.
- Segment click guardrail: clickable preview overlays must not suppress the magnifier. On click, clear the transient magnifier before opening the review editor so the editor is not hidden by the spotlight.
- Spotlight hover guardrail: once the fixed spotlight is visible, resolve the hovered segment from the cursor position inside the spotlight's transformed PDF coordinates before falling back to the underlying page overlays. Otherwise the blue hover outline can appear offset from the section the user is actually pointing at in the enlarged spotlight.
- Smoothness guardrail: do not snap the spotlight's vertical source center to the nearest PDF text line. It may use the nearest line for stable zoom/height context, but the vertical crop must follow raw pointer Y so small cursor moves produce continuous background-position changes instead of a dead zone followed by a jump.
- Verification note: the side-panel hover check used a 900px-wide surface and confirmed the spotlight stretched from x=0 to x=900. The stable-hover check kept `backgroundSize` at `992.8px 1284.8px` across three nearby hover positions while only the vertical position changed.
