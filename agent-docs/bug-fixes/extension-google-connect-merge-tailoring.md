- Symptom: reconnecting the Chrome extension to Google could land on the Google-backed account without the tailoring work that had already started under the prior Job Helper session.
- Root cause: the extension Google auth exchange only sent the Google access token, so the server could not identify the prior bearer session whose active interviews or saved tailored resumes needed to move.
- Fix: send the stored extension bearer token during Google exchange and, when it resolves to a different user than the Google account, merge source Tailor Resume profile entries, active interviews, mirrored Prisma rows, and PDFs into the Google user. Source in-flight or completed tailoring wins over target records for the same job URL, application, run, or tailored-resume id.
- Regression check: start or save tailoring under one extension session, reconnect Google to an account that already has a matching job, then confirm the source active/completed tailoring appears on the Google account and replaces the older target copy.
