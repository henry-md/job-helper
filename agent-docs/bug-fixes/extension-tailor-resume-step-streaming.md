- Symptom: after the extension finished Tailor Resume follow-up question step 2, the side panel could sit on `Thinking...` long enough to look hung even when the backend was still working through steps 3 and 4.
- Root cause: the dashboard requested `x-tailor-resume-stream: 1` and consumed step events, but the extension still waited on a plain JSON `PATCH` response. The backend also only emitted some step updates after work completed, so long model calls had no visible "started step X" signal.
- Fix: extension tailoring actions now request the streamed response, update the side-panel status from live step events, and show inline follow-up progress while finishing. The server now emits step-start events for the long-running stages and logs each step transition with the tailoring run id plus interview id when present.
- Guardrail: any client that can start or resume the Tailor Resume pipeline should consume the streamed step events instead of assuming the route only returns one final JSON payload.
