- Symptom: saved tailored resumes in the extension stopped behaving like first-class Tailor items. The row lost its overflow menu, `Go to tab`, and inline delete action, the archive affordance drifted into a text button, and clicking a saved resume opened the web dashboard instead of the extension's own review/preview detail surface.
- Root cause: the archive split introduced a dedicated saved-resume row renderer that no longer reused the same interaction patterns as active Tailor cards. That renderer kept the dashboard handoff callback, dropped the per-row overflow menu, and no longer fed a saved resume id into the fullscreen Tailor detail state that loads block edits and preview PDFs.
- Fix: keep saved tailored resumes on the extension's native detail path by tracking a selected tailored-resume id, letting fullscreen Tailor detail resolve review and preview state from that id, and restoring per-row actions around the existing background `JOB_HELPER_GO_TO_TAB` handler.
- UI guardrail: saved tailored-resume rows should still expose the same core affordances as active Tailor cards: `Go to tab`, delete from an overflow menu, and an inline archive/restore control.
- UI guardrail: the extension should not punt a saved tailored resume to the web app just to inspect edits or the PDF preview. Review and preview need to remain available directly inside the side panel.
