- Symptom: stopping or deleting an in-flight Tailor Resume run could leave the current-page shell visible for too long, or clear the shell visually while the run still lingered in local extension registries. That stale registry state then made quick recreate flows flaky because the next run could race with old per-page data.
- Root cause: the current-page stop/delete handlers optimistically cleared React state, but they still relied on implicit `readCurrentPageRegistryKey()` lookups when removing `jobHelperTailoringRuns`, `jobHelperTailoringPrompts`, and `jobHelperTailoringPreparations`. In the extension surrogate/browser-check flow, that lookup can drift away from the job tab and miss the real job-page registry entry. Delete also waited too long before clearing local UI.
- Fix: clear in-flight UI immediately, abort the active request immediately, and remove local run/prompt/preparation entries using the captured job registry key derived from the tailoring job URL instead of whatever tab is currently focused. For current-page delete, optimistically remove only in-flight tailoring state unless a saved tailored resume artifact is actually being deleted. Background cards should also optimistically evict their page-key registry entry before awaiting the cancel/delete round-trip.
- Follow-up bug: the preflight phase before a run starts was not actually abortable, so a user could stop/delete a "Checking..." shell, reload, and then watch it come back when the background preflight eventually wrote prompt/run state after the UI had already cleared.
- Follow-up fix: propagate the abort signal through overwrite-summary fetch and add explicit abort checks after each major preflight await before any later state write can repopulate the panel.
- Follow-up bug: background Tailor Resume cards shared one menu action state, so stopping/deleting one run briefly disabled unrelated run cards even though the stopped run was optimistically removed.
- Follow-up fix: track background run stop/delete/go-to-tab action state per card so unrelated active runs remain actionable while each cancel/delete request settles in parallel.
- Guardrail: current-page Tailor Resume cleanup must target the run's own registry key, not ambient tab focus, and stop/delete should leave the panel in a clean "Tailor Current Page" state quickly enough that a recreate can start right away.
