- Symptom: after a tailored resume finished, the extension could still show one active current-page run card even though the web app and synced data already showed that run as completed in the unarchived list.
- Root cause: the extension kept a transient per-page run in `chrome.storage.local` and only retired it when the saved tailored resume matched the current page identity exactly. Some job pages, especially Microsoft listing/detail variants, finished with a saved tailored-resume URL that was comparable to the page run URL but not identical to the current page identity fields.
- Fix: treat a completed unarchived tailored resume with the same comparable URL as sufficient proof that the current-page run is done, suppress the active shell, and immediately clear the transient current-page run from local storage instead of waiting for the stale-run timeout.
