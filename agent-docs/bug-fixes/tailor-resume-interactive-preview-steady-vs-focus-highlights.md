Tailor Resume interactive preview steady vs focus highlights:

- Symptom: the interactive PDF preview could show only a persistent blue focus overlay, fail to paint steady diff highlights, or miss matches entirely when the PDF text layer collapsed spaces or inserted bullet glyphs between items.
- Root cause: preview queries were still using a coarse single-range diff, some rendered plain-text anchors kept layout assignment fragments like `=112pt`, the compact PDF-text matcher treated bullet glyphs as meaningful characters even when the anchor text had no equivalent separator, and focus scrolling cached only the edit token so a card switch could reuse the previous edit's rects before the new rects finished resolving.
- Fix: build interactive preview ranges from the same smarter diff clustering used in the side-by-side review, carry per-range tones for steady yellow/green overlays, strip inline command assignments while rendering LaTeX to preview text, ignore decorative bullet glyphs during compact matching, scroll the actual preview container to the focused diff bounds, key the scroll cache by the resolved focus-rect signature instead of just the edit token, and make the blue guided overlay expire after a short pulse instead of staying mounted while an edit remains selected.
- Guardrail: steady interactive overlays should represent the resolved current diff state, while the blue focus treatment should be temporary guidance only and must not be the only visible highlight once the pulse ends. When focus scrolling changes, target the preview scroll container directly and compute centering from real page/scroller rects instead of relying on overlay `scrollIntoView()` behavior or `offsetTop` guesses.
