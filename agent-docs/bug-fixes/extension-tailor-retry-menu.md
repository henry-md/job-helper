- Symptom: a Tailor Resume job could stall or finish with a result the user wanted to rerun, but the extension menu only exposed stop/delete on active runs and regenerate was limited to the current completed shell.
- Fix: add a consistent `Retry` action to active run cards, the current run shell, and saved tailored-resume row menus. Active retries cancel and clear the in-flight run first, then restart the existing overwrite tailoring flow for the same job URL. Finished retries reuse the same overwrite flow and pass the known application/resume ids when available so the old result can be suppressed and replaced cleanly.
- Guardrail: retry should not use the normal stop tombstone path, because that can hide the new run immediately after it starts. Cancel only the target run, clear its local registry entry, and then dispatch the existing capture/overwrite flow.
