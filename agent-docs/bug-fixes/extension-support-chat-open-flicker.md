- Symptom: opening the extension Resume Chat could briefly flash the empty-chat prompt before showing the loading state. The flash lasted only a frame or two, so it looked like a small jitter rather than a normal state transition.
- Root cause: the launcher only set `isChatOpen`. The support-chat history load was started from a later effect, so React painted the newly mounted chat panel with the previous `idle` status and empty messages before the effect cleared the state and marked it loading.
- Fix: prepare the chat state inside the open action. Signed-in opens now clear old messages, clear errors, and set `chatStatus` to `loading` in the same batched update that mounts the panel; signed-out opens reset to the idle sign-in prompt.
- Adjacent cleanup: signed-out settings/chat resets should be idempotent. Returning a fresh empty array or object from a signed-out effect can create React maximum-update-depth warnings in debug mode because the reset changes one of the effect dependencies every render.
- Guardrail: for transient overlays that immediately fetch data after opening, initialize the visible loading/error/empty state in the user action that opens the surface, not only in a mount effect. Effects can still perform the fetch, but they should not be the first place the visual state becomes accurate.
