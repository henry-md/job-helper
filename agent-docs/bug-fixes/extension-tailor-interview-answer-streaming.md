- Symptom: after answering a Tailor Resume Step 2 follow-up question, the extension could sit on `Thinking...` and then render the assistant's whole tool-call response at once.
- Root cause: the server already supported streamed `interview-stream` events for `advanceTailorResumeInterview`, but the extension did not request the NDJSON stream for that action and did not render a pending assistant message while the answer request was in flight.
- Fix: `advanceTailorResumeInterview` now opts into the streamed Tailor Resume response, and the Resume Questions UI appends a separate optimistic assistant message as `assistantMessage` or `completionMessage` tool-call argument chunks arrive. The standalone thinking dots are hidden once streamed content is visible, while controls stay disabled until the final payload lands.
- Guardrail: every extension action that can trigger Step 2 model output should request and consume `interview-stream` events; otherwise tool-call fields such as `assistantMessage`, `completionMessage`, and `technologyContexts` will look like they arrive all at once.
