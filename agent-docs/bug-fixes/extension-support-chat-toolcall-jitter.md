- Symptom: asking the extension Resume Chat to create a skills-section skill could make the assistant bubble visibly snap. The pending assistant bubble rendered as a tiny thinking card, then expanded when the saved response and tool-call disclosure arrived.
- Root cause: the optimistic assistant message had no content or tool-call disclosure yet, so CSS sized it to the animated dots. The final `done` event replaced the same message with normal text plus `See Toolcalls`, causing a one-frame width and height jump.
- Fix: mark the transient empty assistant response with a `chat-message-thinking` class and reserve the typical saved-skill response footprint while it is streaming. The final saved response can replace the placeholder without resizing the bubble.
- Black-flash follow-up: the chat panel also used a nearly opaque full-screen `backdrop-filter`. During support-chat `done` handling the underlying Tailor tab can refresh skill/profile state, and that backdrop layer can be recomposited as a brief black/underlying-tab flash even though the chat panel never unmounts. Keep the chat panel opaque and isolated instead of relying on backdrop blur.
- Guardrail: optimistic streaming placeholders should reserve the footprint of the most likely terminal state when a tool call may arrive as a single `done` event instead of as gradual text deltas.
